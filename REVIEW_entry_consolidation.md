# 入口の統廃合案（index / v2 / wizard / tutorial / samples）

作成日: 2026-07-17 / 目的: 初心者が「どこから入ればいいか」で迷わない構成にする。

---

## 1. 現状のリンク関係（調査結果）

```
index.html (ルート・フル機能)
  ├─→ tutorial.html   （ヘッダー「はじめての方へ」・別タブ）
  ├─→ samples/        （ヘッダー「完成サンプル」）
  └─→ v2/             （ヘッダー「1つずつ進める版」＋画像カード内リンク）

tutorial.html ──→ index.html（戻る／使ってみる）, v2/（本文リンク）
samples/      ──→ ../index.html（戻る）だけ
v2/           ──→ どこにも出ていく導線が無い（＝入ったら行き止まり）
wizard/       ──→ index.html, v2/（本文）へは出るが、
                  【どこからもリンクされていない＝到達不能な孤児】
```

**判明した3つの構造問題**
1. **wizard は既に孤児**。index のヘッダーは 2026-07 に v2 へ差し替え済みで、wizard へ入るリンクはどこにも無い。DESIGN.md だけが存在を知っている状態。
2. **v2 は出口が無い**。v2 で始めた初心者は samples もフル版も tutorial も見に行けず袋小路。
3. **ステップ版が2つ並存**（旧 wizard・新 v2）。wizard は画像・アイコンにも対応するが到達不能、v2 は LP のみ対応で表に出ている、と機能と露出が逆転している。

## 2. 目標とする構成（あるべき姿）

**「入口は実質2つ＋静的2ページ、初心者の既定は1つだけ」** に整理する。

| 入口 | 役割（再定義） | 対象 |
|---|---|---|
| **v2/**（＝かんたん版） | 初心者の**既定ルート**。1問ずつ進む | はじめての人 |
| **index.html**（＝フル版） | 全項目を1画面で触る作り込み版 | 慣れた人・細かく詰めたい人 |
| samples/ | 完成イメージの参照（静的） | 全員 |
| tutorial.html | 詳しい説明（静的）。index の「これは何？」化で役割を縮小 | 読みたい人だけ |
| ~~wizard/~~ | **削除**（v2 に置き換え済み・孤児） | ― |

## 3. 移行プラン（2段階）

### フェーズ1：今すぐ（低リスク・見た目の入れ替えだけ）

ロジックは触らず、リンクと文言の付け替えで「迷い」を消す。

1. **wizard/ を削除**（または `_archive/wizard/` へ退避）。
   到達不能なので影響ゼロ。DESIGN.md の該当記述も「v2 に統合済み・削除」に更新。
2. **index ヘッダーの3リンクを優劣づけ**。
   - 主ボタン（一番目立つ）＝ **「▶ はじめる（迷ったらこちら）」→ v2/**
   - 副リンク＝「完成サンプル」「使い方」
   - ＋見出し下に1行: 「全項目を1画面で触りたい人向けです。初めてなら『はじめる』へ」
3. **v2 に出口を付ける**（袋小路の解消）。v2 の最後（confirmStep 後）と各画面のどこかに:
   - 「✨ 完成サンプルを見る」→ `../samples/`
   - 「⚙ フル機能版で細かく作る」→ `../index.html`
4. **index に「これは何？」ブロックを常設**（#2の施策）。合わせて **tutorial は“詳しい版”への入口だけ**に縮小し、内容の二重管理を解消。

これでユーザーから見える入口は **v2（既定）／index（フル）／samples／tutorial** の4つだが、**初心者が押す場所は1つ（はじめる→v2）** に定まる。

### フェーズ2：v2 が画像・アイコンに対応したら（最終形）

DESIGN.md ロードマップの「v2 に画像作成・アイコン作成を対応」が済んだ時点で:

1. **ルートを入れ替える**。`v2/index.html` の中身を `index.html`（ルート）に昇格。
2. 現在のフル機能版は `full/`（または `pro/`）へ移動し、「全部を1画面で」派向けの副入口に。
3. tutorial の残りの説明（出力タイプ・業種早見表など）を v2 各ステップの文脈ヒントへ吸収（DESIGN.md 既定の方針）。

## 5. 依存関係（先に決めること）

- **ブロッカー**: v2 は現状 LP のみ。フェーズ2（ルート昇格）は **v2 のタブ対応（画像・アイコン）が前提**。
- それまでの間、画像作成・アイコン作成に到達できるのは index（フル版）だけになる。
  → フェーズ1では index を「画像・アイコンも作れる版」として残す位置づけが自然（消さない）。
- 判断が要る1点: **wizard を「今すぐ削除」か「_archive 退避」か**。v2 が画像・アイコンのステップ版を持つまで、
  ステップ版で画像を作りたい人の唯一の実装は wizard なので、コードの参考として `_archive/` 退避が無難。
