## B. 重い1〜3の具体的な直し方

### 1. 入口を1本化する

**問題の核**: 「通常版」「ステップ版」「旧版」の3つが並存し、初心者が選べない。

**直し方（推奨・段階的）**
- **最優先**: ヘッダーの3リンクを役割で並べ替え、初心者の既定ルートを明示する。
  - 現状: 「はじめての方へ / 完成サンプル / 1つずつ進める版」＝並列で優劣が無い。
  - 案: 一番目立つ主ボタンを **「▶ はじめる（迷ったらこちら）」→ v2** に。「完成サンプル」「フル機能版(index)」は副次リンクに格下げ。
- **中期**: DESIGN.md の方針どおり、初回訪問者は v2 に着地させる。index は「全部を1画面で触りたい人向け」と明記して上級者ルート化。
- **掃除**: `wizard/` は導線を外して温存中とのことなので、リンクが本当にどこからも張られていないか確認し、残っていれば削除（重複入口を1つ減らせる）。

**文言例（v2 or index ヘッダー直下に1行）**
> 初めてなら「はじめる」だけでOK。細かく触りたくなったら上のタブや詳細設定へ。

---

### 2. 本体(index)に「これは何？」を1ブロック足す

**問題の核**: 説明が tutorial.html に隔離され、直接来た人に届かない。「LP」の定義も無い。

**直し方**
- index のヘッダー直下（プリセットカードの前）に、tutorial の「これは何のツール？」を**3行に圧縮**して常設。折りたたみ不要、常時表示。
- 冒頭で **LP を1回だけ定義**する。

**文言例（そのまま貼れる）**
```html
<div class="card card--intro">
  <p><b>LP（ランディングページ）</b>＝商品やお店を1ページで紹介する縦長のWebページのことをLPと呼びます。</p>
  <p>このツールは「何をどの順番で書くか」の<b>設計図</b>を、業種・読み手・ゴールを選ぶだけで作ります。</p>
  <p>できた文章を <b>ChatGPT や Claude に貼る</b>と、原稿やLPそのものに変わります（このツール自体は無料・AI不要）。</p>
</div>
```
- 同じ内容が tutorial にもあるので、**tutorial 側は「詳しい版」への入口だけ残す**と重複が消える（内容の二重管理を避ける）。

---

### 3. 専門用語に「その場で分かる」補足を付ける

**問題の核**: 未知語が説明なしで並ぶ。別ページに飛ばすと離脱する。

**直し方（離脱させない順）**
1. **言い換え優先**: 消せる専門語はやさしい語へ。
   - 「見取り図」→ 併記で「（LPの形のプレビュー）」
   - 「出力タイプ」→「作りたいものを選ぶ」
   - 「プロンプト」→「AIへの指示文」（初出のみ括弧で併記）
2. **WebP / ZIP** は結果だけ伝える: 「軽い画像に自動変換」「1つのファイルにまとめて保存」など、名称より効果を書く。
3. **GitHub / git init（v2ステップ0）** は初心者の必須路から外す。
   - 現状「読まなくて大丈夫」でも心理的ハードル。**既定を「スキップして進む」にして**、リポジトリ準備は「あとで公開したくなったら」の任意ステップへ後退させる。
   - もしくはステップ0自体を、プリセット選択の**後ろ**に移動（まず成果物を見せてから、公開の話をする）。
4. 用語を残す場合は `<abbr title="...">` かツールチップで**ホバー説明**を付け、ページ遷移ゼロで理解できるようにする。
